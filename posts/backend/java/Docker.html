Docker相关

架构

docker build 构建镜像
docker pull 从仓库拉取镜像
docker run 运行镜像

安装（debian）

详见:

https://docs.docker.com/engine/install/debian/

  1. 卸载冲突包
for pkg in docker.io docker-doc docker-compose podman-docker containerd runc; do sudo apt-get remove $pkg; done
  1. 设置docker的apt存储库
# Add Docker's official GPG key:
sudo apt-get update
sudo apt-get install ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/debian/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc

# Add the repository to Apt sources:
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/debian \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt-get update
  1. 安装docker包
sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
  1. 验证
sudo docker run hello-world

此命令下载测试映像并在容器中运行。容器运行时，它会打印一条确认消息并退出。

  1. 更换镜像

可以通过修改daemon配置文件/etc/docker/daemon.json来使用加速器

sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": ["https://qnmufcit.mirror.aliyuncs.com"]
}
EOF
sudo systemctl daemon-reload
sudo systemctl restart docker

常见命令

基本操作

  1. 查看 Docker 版本
docker --version
  1. 查看 Docker 系统信息
docker info
  1. 列出所有 Docker 命令
docker --help

镜像管理

  1. 搜索镜像
docker search [镜像名]
  1. 拉取镜像
docker pull [镜像名]:[标签]
  1. 列出本地镜像
docker images
  1. 删除本地镜像
docker rmi [镜像ID]
  1. 构建镜像
docker build -t [镜像名]:[标签] [Dockerfile路径]

容器管理

  1. 运行容器
docker run [选项] [镜像名]
-d //后台运行
-t //分配伪终端
-i //交互模式
-it //交互模式和伪终端
--name container_name //指定容器名称
-p host_port:container_port // 端口映射
-e "ENV_VAR_NAME=value" //环境变量 可覆盖ENV中同名变量
-v /host/path:/container/path 挂载券
-w /path/in/container //设置工作目录
--memory="1g" //设置工作内存
--cpus="0.5" 限制cpu数量
--rm //容器停止后自动删除
  1. 列出运行中的容器
docker ps
  1. 列出所有容器（包括停止的）
docker ps -a
  1. 停止容器
docker stop [容器ID]
  1. 启动容器
docker start [容器ID]
  1. 重启容器
docker restart [容器ID]
  1. 删除容器
docker rm [容器ID]
  1. 查看容器日志
docker logs [容器ID]
  1. 进入容器
docker exec -it [容器ID] /bin/bash

网络管理

  1. 列出网络
docker network ls
  1. 创建网络
docker network create [网络名]
  1. 删除网络
docker network rm [网络名]
  1. 连接容器到网络
docker network connect [网络名] [容器ID]
  1. 断开容器与网络的连接
docker network disconnect [网络名] [容器ID]

卷管理

  1. 列出卷
docker volume ls
  1. 创建卷
docker volume create [卷名]
  1. 删除卷
docker volume rm [卷名]
  1. 查看卷详细信息
docker volume inspect [卷名]

使用Dockerfile构建镜像

流程

  1. 创建一个目录
mkdir my-docker-app
cd my-docker-app
  1. 创建Dockerfile
touch Dockerfile
  1. 编写dockerfile
  2. 构建镜像
docker build -t my-python-app .
-t 指定tag标签
. 表示Dockerfile所在的当前目录

常用指令

Dockerfile 是用来定义 Docker 镜像构建过程的文件。以下是一些常用的 Dockerfile 指令及其详细解释。

  1. FROM
FROM <image>[:<tag>]
# 指定基础镜像，通常是 Dockerfile 的第一条指令。 
FROM python:3.9-slim
  1. LABEL
LABEL <key>=<value> ...
# 添加元数据到镜像中，例如维护者信息。
LABEL maintainer="you@example.com"
  1. RUN
RUN <command> [param1 param2 ...]
# 在构建镜像时执行的命令。通常用于安装软件包。
RUN apt-get update && apt-get install -y curl
  1. CMD
CMD ["executable","param1","param2"]
# 指定容器启动时要运行的命令。可以被 docker run 命令行参数覆盖。
CMD ["python", "app.py"]
  1. EXPOSE
EXPOSE <port> [<port>/<protocol>...]
# 声明容器运行时监听的端口。
EXPOSE 80
  1. ENV
ENV <key>=<value> ...
# 设置环境变量。
ENV DEBUG=true
  1. ADD
ADD <src>... <dest>
# 将文件、目录或远程文件 URL 的内容复制到镜像中。
ADD . /app
  1. COPY
COPY <src>... <dest>
# 将文件和目录复制到镜像中（与 ADD 类似，但不支持 URL）。
COPY . /app
  1. ENTRYPOINT
ENTRYPOINT ["executable", "param1", "param2"]
# 配置容器启动时运行的主命令，不会被 docker run 命令行参数覆盖。
ENTRYPOINT ["python", "app.py"]
  1. VOLUME
VOLUME [""/path/in/container""]
# 创建挂载点，将主机目录或其他容器目录挂载到容器中。
VOLUME ["/myvol"]
  1. WORKDIR
WORKDIR /path/to/workdir
# 设置工作目录，用于 RUN, CMD, ENTRYPOINT, COPY 和 ADD 指令。
WORKDIR /app
  1. USER
USER <username|uid>[:<group|gid>]
# 设置执行后续指令的用户和可选的用户组。
USER appuser
  1. ARG
ARG <name>[=<default value>]
# 定义在构建时可以传递的变量。
ARG build_version
  1. ONBUILD
ONBUILD <INSTRUCTION>
# 设置触发指令，当构建一个继承自这个镜像的镜像时执行。
ONBUILD ADD . /app/src

例

# 使用官方的 Node.js 镜像作为基础镜像
FROM node:14

# 设置维护者信息
LABEL maintainer="you@example.com"

# 设置环境变量
ENV NODE_ENV=production

# 设置工作目录
WORKDIR /usr/src/app

# 将 package.json 和 package-lock.json 复制到工作目录
COPY package*.json ./

# 安装应用依赖
RUN npm install

# 将应用代码复制到工作目录
COPY . .

# 暴露应用运行端口
EXPOSE 8080

# 定义容器启动时运行的命令
CMD ["node", "server.js"]
